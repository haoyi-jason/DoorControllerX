# DoorControllerX 使用者操作手冊

## 現場一頁版 SOP + 錯誤碼速查

### A. 快速操作 SOP（現場流程）
1. 開啟 Host，選擇 `Endpoint`，確認 `Baud=115200`，按 `Connect`。
2. 按 `Read Snapshot`，確認 `Error Code=0`。
3. 按 `Start Poll` 進入持續監看。
4. 執行需要的動作：`開門` / `關門` / `上鎖` / `解鎖`。
5. 觀察重點：`System State`、`Main/Sub Door Angle`、`Operation Time`、`Error Code`。
6. 若報錯：先排除機構或感測異常，再按 `清除錯誤`（或控制器端長按 TEST > 3 秒）。
7. 若要連續壓測：到 `Parameters` 將 `DF_AUTO_TEST_CYCLES` 設定為 `1~200`；測試完成後務必寫回 `0`。

### B. 錯誤碼速查表（Error Code）
| 錯誤碼 | 說明 | 建議處置 |
| --- | --- | --- |
| 0 | 無錯誤 | 正常操作 |
| 1 | 阻擋重試超過上限 | 清除障礙物、檢查門扇阻力，再清錯重試 |
| 2 | 操作逾時（超過最大操作時間） | 檢查機構卡滯、行程與參數設定 |
| 3 | 執行中解鎖失敗 | 檢查電鎖機構、解鎖方向與感測訊號 |
| 4 | 執行中上鎖失敗 | 檢查電鎖機構、上鎖方向與感測訊號 |
| 10 | 上電解鎖方向/感測檢查失敗 | 檢查電鎖方向、線序、感測器對應 |
| 11 | 副門 VR 與運轉方向不一致 | 檢查副門方向設定、馬達線序、感測方向 |
| 12 | 副門回原點逾時 | 檢查副門機構卡滯、原點開關與參數 |
| 13 | 副門原點開關異常 | 檢查副門原點開關與配線 |
| 14 | 主門 VR 與運轉方向不一致 | 檢查主門方向設定、馬達線序、感測方向 |
| 15 | 主門回原點逾時 | 檢查主門機構卡滯、原點開關與參數 |
| 16 | 主門原點開關異常 | 檢查主門原點開關與配線 |
| 17 | 上電上鎖方向/感測檢查失敗 | 檢查電鎖方向、線序、感測器對應 |

### C. 三個高頻問題
- 連不上裝置：先 `Refresh`，確認 COM/藍牙端點與 `115200`。
- 清錯後立刻再報錯：代表根因未排除，請先檢查機構與感測器。
- 自動測試停不下來：將 `DF_AUTO_TEST_CYCLES` 寫回 `0`。

## 適用對象
- 現場操作員
- 維護工程師
- 調機與測試人員

## 適用範圍
本手冊說明 `DoorControllerX Host` 應用程式的基本操作方式，適用於：
- Windows 版本
- Android 版本

本手冊聚焦於實際操作，不說明底層通訊協定與韌體設計細節。

## 系統用途
DoorControllerX 用於控制雙門與電鎖機構，主要功能包含：
- 監看主門與副門狀態
- 監看角度、錯誤碼、重試次數、操作時間
- 遠端執行開門、關門、上鎖、解鎖
- 清除錯誤狀態
- 讀寫控制參數（DF Parameters）
- 執行自動開門/關門循環測試

## 操作前準備
使用前請先確認：
- 控制器已上電
- 主機與控制器的通訊線路或藍牙模組正常
- Windows 已確認對應的 COM Port
- Android 已完成藍牙配對
- 門扇運動範圍內無人員與障礙物
- 若要執行自動測試，請先確認機構可安全連續運轉

## 主畫面說明
主畫面主要分為以下區域：

### 1. Connection 區
用途：建立與控制器的連線。

欄位與按鈕：
- `Endpoint`
  - Windows: 通常是 `COMx`
  - Android: 通常是藍牙裝置名稱或 MAC 位址
- `Manual`
  - 可手動輸入端點，例如 `COM5`
- `Baud`
  - 預設為 `115200`
- `Refresh`
  - 重新搜尋可用端點
- `Connect`
  - 建立連線
- `Disconnect`
  - 中斷連線
- `Read Snapshot`
  - 立即讀取一次目前狀態
- `Start Poll` / `Stop Poll`
  - 開始或停止週期性讀取資料
- `Parameters`
  - 進入參數設定頁

### 2. 遠端控制區
用途：直接對控制器下達操作命令。

按鈕說明：
- `開門`
- `關門`
- `上鎖`
- `解鎖`
- `清除錯誤`

注意：
- 未連線時，按鈕不可用
- 執行遠端命令前，請先確認現場安全
- `清除錯誤` 只會清除錯誤狀態，不會自動排除機構本身故障

### 3. 狀態監看區
用途：顯示目前控制器狀態與門扇資訊。

主要欄位：
- `System State`
  - 系統狀態
- `Main Door State`
  - 主門狀態
- `Sub Door State`
  - 副門狀態
- `Main Door Angle`
  - 主門角度
- `Sub Door Angle`
  - 副門角度
- `Block Count`
  - 阻擋偵測次數
- `Block Retry Count`
  - 阻擋後自動重試次數
- `Block Source State`
  - 最後一次阻擋發生在哪個流程狀態
- `Main Door Raw Angle`
  - 主門原始電位器角度
- `Sub Door Raw Angle`
  - 副門原始電位器角度
- `Close Stage`
  - 雙門關門流程階段
- `Error Code`
  - 目前錯誤碼
- `Lock Retry Count`
  - 電鎖重試次數
- `Operation Time`
  - 當前動作執行時間
- `Open Count`
  - 累積開門次數
- `Close Count`
  - 累積關門次數
- `Reset Reason`
  - 最近一次重置原因

## 基本操作流程

### 建立連線
1. 啟動 `DoorControllerX Host`
2. 在 `Endpoint` 下拉選單選擇裝置，或於 `Manual` 手動輸入端點
3. 確認 `Baud = 115200`
4. 按下 `Connect`
5. 連線成功後，按鈕會切換為可執行狀態
6. 可按 `Read Snapshot` 讀一次即時狀態
7. 若要持續監看，按下 `Start Poll`

### 中斷連線
1. 按下 `Disconnect`
2. 系統將停止輪詢與圖表採樣

## 遠端控制操作

### 開門
1. 確認控制器已連線
2. 確認門扇前方無障礙物
3. 按下 `開門`
4. 觀察 `System State`、`Main Door State`、`Sub Door State` 與角度變化

### 關門
1. 確認門扇運動路徑安全
2. 按下 `關門`
3. 觀察關門流程與最終狀態

### 上鎖 / 解鎖
1. 確認機構允許目前動作
2. 按下 `上鎖` 或 `解鎖`
3. 觀察錯誤碼、重試次數與狀態是否正常

### 清除錯誤
錯誤發生後，可用以下兩種方式清除：
- 在 Host 畫面按下 `清除錯誤`
- 在控制器端長按 `TEST` 超過 3 秒

使用時機：
- 已確認異常原因排除後
- 需要讓系統回到待機狀態時

注意：
- 若機構故障仍存在，清除後仍可能再次報錯

## Parameters 頁面操作
按主畫面的 `Parameters` 可進入參數頁。

頁面內容：
- `Select DF parameter`
  - 選擇參數
- `ID`
  - 參數編號
- `Range`
  - 可設定範圍
- `Default`
  - 預設值
- `Value`
  - 要讀或要寫入的值
- `Read`
  - 讀取目前值
- `Write`
  - 寫入目前值
- `Read All`
  - 讀取全部參數
- `Back`
  - 返回主畫面

### 讀取單一參數
1. 進入 `Parameters`
2. 於下拉選單選擇參數
3. 按 `Read`
4. 目前值會顯示在 `Value`

### 修改單一參數
1. 選擇要修改的參數
2. 先確認 `Range`
3. 在 `Value` 輸入新值
4. 按 `Write`
5. 建議寫入後再按一次 `Read` 確認

### 讀取全部參數
1. 按 `Read All`
2. 內容會顯示在 `Read All Result`

## 自動開門/關門測試
本系統支援用參數設定自動循環測試次數。

相關參數：
- `DF_AUTO_TEST_CYCLES`

意義：
- `0`: 停用自動測試
- `1~200`: 設定自動開門/關門循環次數

### 操作方式
1. 進入 `Parameters`
2. 選擇 `DF_AUTO_TEST_CYCLES`
3. 輸入測試次數，例如 `10`
4. 按 `Write`
5. 返回主畫面
6. 觀察系統自動執行開門與關門流程

### 測試期間建議觀察項目
- `System State`
- `Main Door Angle`
- `Sub Door Angle`
- `Error Code`
- `Operation Time`
- `Open Count`
- `Close Count`

### 停止方式
- 將 `DF_AUTO_TEST_CYCLES` 寫回 `0`
- 若系統進入錯誤狀態，先排除原因後再按 `清除錯誤`

## 即時圖表操作
主畫面提供即時圖表，用來觀察 Live Data 或 DF 參數的變化。

可操作項目：
- 設定時間窗（秒）
- 選擇 Y1 / Y2 自動縮放或手動範圍
- 設定最多 4 條曲線
- 指定每條曲線資料來源與使用的軸
- `Clear Chart` 清除圖表資料

### 建議用途
- 觀察門角度變化
- 觀察 PWM 與角度關係
- 進行調機與 PID 試算前的趨勢分析

## PID Auto Tuning（試算）
主畫面提供 `Estimate PI` 功能，可針對圖表曲線做 PI 參數試算。

建議流程：
1. 先選擇一條有效曲線
2. 輸入 `Setpoint`
3. 選擇調整目標
4. 按 `Estimate PI`
5. 參考試算結果作為調機起點

注意：
- 這是試算工具，不保證可直接套用
- 實際參數仍需依機構特性微調

## 錯誤處理

### 錯誤發生時的建議步驟
1. 先查看 `Error Code`
2. 觀察是否伴隨 `Block Count`、`Lock Retry Count`、`Operation Time` 增加
3. 檢查門扇、電鎖、感測器與機構是否卡住
4. 排除異常後，按 `清除錯誤`
5. 再重新測試一次

### 常見錯誤來源
- 門扇阻擋
- 電鎖無法到位
- 感測器訊號異常
- 啟動或動作逾時
- 機構卡死或安裝偏移

## 建議日常操作流程

### 一般監看
1. 連線
2. `Read Snapshot`
3. `Start Poll`
4. 觀察狀態與錯誤碼

### 單次功能測試
1. 連線
2. 先確認 `Error Code = 0`
3. 按 `開門` 或 `關門`
4. 觀察角度、狀態與操作時間

### 維修後驗證
1. 連線並讀取快照
2. 確認無錯誤
3. 執行手動開關門
4. 必要時啟用 `DF_AUTO_TEST_CYCLES` 進行連續測試
5. 確認整體運作穩定後，將 `DF_AUTO_TEST_CYCLES` 設回 `0`

## 注意事項
- 遠端控制前，務必確認門扇運動範圍安全
- 不要在未確認機構狀態下連續下達多個控制命令
- 修改 DF 參數前，先記錄原始值
- 若參數寫入後行為異常，應立即恢復原設定
- 自動測試不可在有人通行或機構未固定完成時執行

## 快速問答

### 連不上裝置怎麼辦？
- 按 `Refresh`
- 確認端點名稱或 COM Port 是否正確
- 確認鮑率為 `115200`
- 確認藍牙已配對或序列埠未被其他程式占用

### 為什麼按了命令沒有動作？
- 先確認已連線
- 檢查是否處於錯誤狀態
- 查看 `Error Code`
- 確認控制器端與機構供電正常

### 清除錯誤後又立刻再次報錯怎麼辦？
- 代表根本原因尚未排除
- 請先檢查門扇、電鎖、感測器與安裝狀態

### 自動測試怎麼關掉？
- 將 `DF_AUTO_TEST_CYCLES` 寫回 `0`
